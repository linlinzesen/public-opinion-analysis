# 网络舆情智能分析功能——前端开发对接文档

> 适用技术栈：Vue 3 + Element Plus + ECharts  
> 后端模块：`llm_service`  
> 独立联调地址：`http://localhost:5001`

## 1. 接口清单

| 功能 | 方法 | 地址 |
|---|---|---|
| 智能问答 | POST | `/api/llm/ask` |
| 未来24小时趋势预测 | POST | `/api/llm/predict` |
| Markdown分析报告 | POST | `/api/llm/report` |

所有接口使用 JSON 和 UTF-8：

```text
Content-Type: application/json; charset=utf-8
```

开发阶段后端可使用 mock 模式，不需要大模型 API Key。

## 3. 数据结构

```ts
export interface TrendPoint {
  time: string
  value: number
}

export interface EventData {
  id?: number | string
  title: string
  summary: string
  keywords?: string[]
  sentiment_distribution?: {
    positive?: number
    neutral?: number
    negative?: number
  }
  trend_data?: TrendPoint[]
  platform_distribution?: Record<string, number>
}
```

事件示例：

```json
{
  "id": 1,
  "title": "某品牌产品质量争议",
  "summary": "消费者发布产品质量问题视频，品牌方随后回应并启动调查。",
  "keywords": ["产品质量", "品牌回应"],
  "sentiment_distribution": {
    "positive": 15,
    "neutral": 30,
    "negative": 55
  },
  "trend_data": [
    {"time": "2026-07-09T08:00:00+08:00", "value": 120},
    {"time": "2026-07-09T09:00:00+08:00", "value": 145},
    {"time": "2026-07-09T10:00:00+08:00", "value": 180}
  ],
  "platform_distribution": {
    "微博": 55,
    "抖音": 30,
    "新闻": 15
  }
}
```

注意：

- 时间使用 ISO 8601；
- 热度必须是数字；
- 趋势预测至少需要两个数据点；
- 情感和平台数据建议统一使用百分数。

## 4. Axios封装

创建 `src/api/llm.ts`：

```ts
import axios from 'axios'

const api = axios.create({
  baseURL: import.meta.env.VITE_API_BASE_URL,
  timeout: 45_000,
  headers: {
    'Content-Type': 'application/json; charset=utf-8'
  }
})

export function askEventQuestion(eventData: any, question: string) {
  return api.post('/api/llm/ask', {
    event_data: eventData,
    question
  })
}

export function predictEventTrend(trendData: any[]) {
  return api.post('/api/llm/predict', {
    trend_data: trendData
  })
}

export function generateEventReport(eventData: any) {
  return api.post('/api/llm/report', {
    event_data: eventData
  })
}
```

`.env.development`：

```env
VITE_API_BASE_URL=http://localhost:5001
```

修改环境变量后需重启 `npm run dev`。

## 5. 智能问答

### 请求

```json
{
  "event_data": {
    "title": "某品牌产品质量争议",
    "summary": "消费者发布质量问题视频，品牌方启动调查。",
    "keywords": ["产品质量", "品牌回应"],
    "sentiment_distribution": {
      "positive": 15,
      "neutral": 30,
      "negative": 55
    },
    "platform_distribution": {"微博": 55, "抖音": 30}
  },
  "question": "该事件当前主要风险是什么？"
}
```

### 响应

```json
{
  "success": true,
  "answer": "当前可从热度变化和负面情感占比研判该事件风险……",
  "mode": "mock",
  "model": "mock-rule-based"
}
```

### Vue调用

```ts
const question = ref('')
const answer = ref('')
const asking = ref(false)

async function submitQuestion() {
  if (!question.value.trim()) {
    ElMessage.warning('请输入问题')
    return
  }

  asking.value = true
  try {
    const { data } = await askEventQuestion(
      currentEvent.value,
      question.value.trim()
    )
    answer.value = data.answer ?? ''
  } catch (error: any) {
    ElMessage.error(error.response?.data?.error ?? '智能问答服务调用失败')
  } finally {
    asking.value = false
  }
}
```

`mode`说明：

- `api`：真实大模型回答；
- `mock`：演示模式回答；
- `guard`：问题与事件无关，仍应正常展示 `answer`；
- `fallback`：真实API失败，后端已生成降级回答。

## 6. 趋势预测

### 请求

```json
{
  "trend_data": [
    {"time": "2026-07-09T08:00:00+08:00", "value": 120},
    {"time": "2026-07-09T09:00:00+08:00", "value": 145},
    {"time": "2026-07-09T10:00:00+08:00", "value": 180}
  ]
}
```

### 响应

```json
{
  "success": true,
  "method": "linear_regression",
  "trend": "上升",
  "change_ratio": 3.9907,
  "interval_hours": 1.0,
  "predictions": [
    {"time": "2026-07-09T11:00:00+08:00", "value": 208.33}
  ]
}
```

小时数据实际返回24个预测点。`change_ratio=0.15`表示上升约15%。

### Vue调用

```ts
const predictions = ref<any[]>([])
const trendDirection = ref('')

async function loadPrediction() {
  const points = currentEvent.value.trend_data ?? []
  if (points.length < 2) {
    ElMessage.warning('至少需要两个热度数据点')
    return
  }

  try {
    const { data } = await predictEventTrend(points)
    predictions.value = data.predictions
    trendDirection.value = data.trend
  } catch (error: any) {
    ElMessage.error(error.response?.data?.error ?? '趋势预测失败')
  }
}
```

### ECharts配置

```ts
const option = computed(() => ({
  tooltip: { trigger: 'axis' },
  legend: { data: ['历史热度', '预测热度'] },
  xAxis: { type: 'time' },
  yAxis: { type: 'value', min: 0 },
  series: [
    {
      name: '历史热度',
      type: 'line',
      data: (currentEvent.value.trend_data ?? [])
        .map((p: any) => [p.time, p.value]),
      smooth: true
    },
    {
      name: '预测热度',
      type: 'line',
      data: predictions.value.map(p => [p.time, p.value]),
      smooth: true,
      lineStyle: { type: 'dashed' },
      itemStyle: { color: '#E6A23C' }
    }
  ]
}))
```

历史线建议使用实线，预测线使用橙色虚线。

## 8. 跨域代理

浏览器出现 `CORS policy` 时，在 `vite.config.ts` 配置：

```ts
import { defineConfig } from 'vite'
import vue from '@vitejs/plugin-vue'

export default defineConfig({
  plugins: [vue()],
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:5001',
        changeOrigin: true
      }
    }
  }
})
```

使用代理时Axios的`baseURL`可改为 `/`。修改代理后必须重启前端。

## 9. 状态码

| 状态码 | 含义 | 前端处理 |
|---|---|---|
| 200 | 成功，含mock/guard/fallback | 正常展示 |
| 400 | 参数或数据格式错误 | 展示响应中的`error` |
| 404 | 地址配置错误 | 检查baseURL和代理 |
| 500 | 服务异常 | 展示统一错误提示 |

## 10. 集成主后端

主Flask应用注册：

```python
from llm_service import create_llm_blueprint

app.register_blueprint(create_llm_blueprint())
```

如果主后端为 `http://localhost:5000`，前端只需修改：

```env
VITE_API_BASE_URL=http://localhost:5000
```

三个接口路径保持不变。
